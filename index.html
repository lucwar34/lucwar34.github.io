How I Learn

My approach to learning is simple: build something, fix inevitable issues, and document what I learned. Through this process, I completed three hands-on projects that were both practically useful in my own environment and directly applicable to real-world IT concepts.

Projects Overview

Virtualization & Linux Command Line

My exposure to virtualization began in a Computer Operating Systems course, where we were required to run a virtual machine hosted on a remote computer lab provided by the university. The process involved accessing the lab environment, installing Ubuntu, and configuring virtual machine specifications using VirtualBox.

Once the virtual machine was configured, much of our interaction took place through the Linux command line. Because the virtual machines were hosted remotely, I continue to access my VM by remotely connecting to a school computer and launching it from there whenever I want to experiment or practice.

This experience later proved valuable in a C Programming for Engineers course, where I worked extensively in a Unix-based terminal to navigate directories, compile code, and execute programs.

TrueNAS & Personal Storage Server

To gain experience with storage and server administration, I repurposed an old laptop by installing TrueNAS as a dedicated operating system. The system is configured and managed remotely through the TrueNAS web interface.

The server is used for centralized storage of photos, game ROM and ISO backups, media streaming via Jellyfin, and hosting a Minecraft server for friends. To maintain consistent access, I configured a static network address at the router level.

I plan to expand this setup by adding a second hard drive for data mirroring and exploring additional virtualization capabilities as resources allow.

Pi-hole & Network-Wide DNS Filtering

To further explore networking concepts, I implemented Pi-hole as a network-wide DNS filtering solution. This involved configuring DNS settings at the router level and managing clients through the Pi-hole interface.

By handling DNS requests centrally, Pi-hole blocks ads and tracking domains across all devices on the network without requiring per-device configuration.
