Virtualization & Linux Command Line

My exposure to virtualization began in a Computer Operating Systems course, where we were required to run a virtual machine hosted on a remote computer lab provided by the university. The process involved accessing the lab environment, installing Ubuntu, and configuring virtual machine specifications using VirtualBox.

Once the virtual machine was configured, much of our interaction took place through the Linux command line. Because the virtual machines were hosted remotely, I continue to access my VM by remotely connecting to a school computer and launching it from there whenever I want to experiment or practice.

This experience became particularly useful in a later C Programming for Engineers course, where I was required to use a Unix-based terminal environment to navigate directories, compile programs, and execute code. Having already spent time working in Ubuntu and the Linux CLI made the transition to the macOS terminal (which is Unix-based) far more intuitive.

TrueNAS & Personal Storage Server

To gain experience with storage and server administration, I repurposed an old laptop by installing TrueNAS as a dedicated operating system. This required identifying compatible hardware, installing a new OS from scratch, and configuring the system remotely from another computer using TrueNAS’s web interface.

With storage hardware becoming increasingly expensive in recent months, particularly due to shortages affecting memory and related production, using an existing one-terabyte hard drive provided a practical and cost-effective solution. The system now functions as a centralized storage server within my home network, all while serving to be a cunning solution to re-use old hardware.

The server is currently used to store personal photos, game ROM files and ISO backups, and a media library accessible through Jellyfin. I have also hosted a Minecraft game server on the system, allowing friends and I to connect and play within a self-hosted environment.

From a technical standpoint, maintaining consistent network access to the TrueNAS server required configuring a static address at the router level to preserve DNS resolution. I plan to expand this setup by adding a second hard drive to mirror the existing one for improved data redundancy, as well as exploring additional virtualization capabilities as my budget allows.

Beyond the technical learning, this project demonstrated the real-world practicality of self-hosted services by reducing reliance on subscription-based cloud storage solutions.

Pi-hole & Network-Wide DNS Filtering

To further explore networking and DNS concepts, I implemented Pi-hole as a network-wide DNS filtering solution. This involved configuring the Pi-hole service, managing clients and settings, and updating router-level DNS configurations to route traffic through the Pi-hole instance.

By handling DNS requests at the network level, Pi-hole blocks advertisements and tracking domains across all connected devices without requiring per-device configuration. Managing this system provided insight into how DNS operates within a network and how centralized services can improve both performance and security.
